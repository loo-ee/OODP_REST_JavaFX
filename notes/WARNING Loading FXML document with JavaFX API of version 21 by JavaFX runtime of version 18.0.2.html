To put Parent root = FXMLLoader into a singleton class and make it work, you need to follow the Singleton design pattern. The Singleton pattern restricts a class to instantiate its multiple objects. It is used where only a single instance of a class is required to control actions.

Here's how you can create a Singleton class that loads an FXML file:

import javafx.fxml.FXMLLoader;
import javafx.scene.Parent;
import java.io.IOException;

public class FXMLLoaderSingleton {
   private static FXMLLoaderSingleton instance;
   private Parent root;

   private FXMLLoaderSingleton() {
       try {
           root = FXMLLoader.load(getClass().getResource("file.fxml"));
       } catch (IOException e) {
           e.printStackTrace();
       }
   }

   public static FXMLLoaderSingleton getInstance() {
       if (instance == null) {
           instance = new FXMLLoaderSingleton();
       }
       return instance;
   }

   public Parent getRoot() {
       return root;
   }
}

In this code:

  • The FXMLLoaderSingleton class is defined with a private constructor to prevent direct instantiation.
  • The instance variable is declared as a private static variable of the class. This variable will hold the single instance of the class.
  • The getInstance() method checks if the instance variable is null. If it is, it creates a new FXMLLoaderSingleton object and assigns it to the instance variable. If the instance variable is not null, it simply returns the existing instance.
  • The getRoot() method returns the Parent object loaded from the FXML file.

You can use this Singleton class in your application like this:

Parent root = FXMLLoaderSingleton.getInstance().getRoot();

This will return the Parent object loaded from the FXML file in the FXMLLoaderSingleton instance geeksforgeeks.org.
